# ADR: Use Grapheme Clusters For Character Collection

## Status

Accepted for Phase 3.

## Context

Character input files may contain ASCII, Japanese text, symbols, and emoji. Byte iteration and Unicode scalar iteration can split user-visible emoji sequences, especially when variation selectors are involved.

Phase 3 needs deterministic character collection before font rasterization exists.

## Decision

Use the `unicode-segmentation` crate and collect Unicode grapheme clusters as display units.

Collection rules:

- Read each configured character file as UTF-8 text.
- Visit files in configured order.
- Remove duplicate display units while preserving first occurrence order.
- Ignore newline, carriage return, and tab.
- Preserve normal spaces only when `preserve_space` is enabled.
- Report an error when no display units remain after filtering.

## Consequences

Emoji variation selectors stay attached to their base sequence when represented as one grapheme cluster.

Font glyph support remains a later validation step. Phase 4 must decide how to report display units that cannot be mapped or rasterized by the selected font library.
