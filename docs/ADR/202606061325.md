# ADR: Document The C Fixed Runtime Contract In README

## Status

Accepted for Phase 9 documentation.

## Context

The generated `c-fixed` header exposes all data needed by rendererless firmware, but the README only summarizes the format. A user currently has to inspect the generated header, source implementation, or minimal test example to determine how UTF-8 lookup, bitmap indexing, nibble unpacking, and display drawing fit together.

## Decision

Add a platform-neutral C integration walkthrough to README.

The walkthrough will:

- Use the generated mapping and bitmap symbols directly.
- Parse UTF-8 display units instead of indexing bytes as characters.
- Keep mapping positions and bitmap indices aligned.
- Decode high-nibble-first 4-bit alpha.
- Delegate final pixel output to a firmware-provided callback.
- Demonstrate integer alpha blending without heap allocation.

## Consequences

- Firmware developers can integrate generated output from README alone.
- The example remains portable across ESP32, Arduino, and Raspberry Pi Pico W projects.
- Display-driver setup, clipping, caching, and performance tuning remain the firmware project's responsibility.
- Multi-scalar grapheme clusters remain unsupported by the current generator.
