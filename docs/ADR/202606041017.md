# ADR 202606041017: Use Config File Plus Character Lists

## Status

Accepted for planning. Implementation is pending user approval.

## Context

The tool needs to generate firmware font data from project-specific characters. The initial example uses one text file containing all characters. The tool also needs generation settings such as font size, output language, source font path, and output name.

Putting generation settings inside the same file as characters would make the character list harder to reuse. Putting every setting in CLI arguments would make repeated generation more error-prone.

## Decision

Use a TOML configuration file for generation settings and one or more plain UTF-8 text files for character lists.

The default configuration file is `fontgen.toml`.

Character files remain plain text and contain only user-facing characters.

CLI arguments may override common settings such as sizes, output language, font path, and character files.

## Consequences

Benefits:

- Character lists can be reused across output languages and font sizes.
- Multiple font sizes can be generated from the same character data.
- CLI commands stay short for normal use.
- One-off overrides remain possible.

Tradeoffs:

- Users need to maintain one config file in addition to character files.
- The tool must clearly define path resolution behavior.

## Follow-Up

Phase 2 must implement config parsing and CLI override merging.

Phase 3 must confirm Unicode grapheme handling and document any rasterization limitations.
